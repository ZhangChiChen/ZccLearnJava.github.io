# ·Java代码规范

1.  类、方法的注释，要以javadoc的方式来写

2. 非javadoc的注释，往往是非代码的维护者看的，着重告诉读者为什么这样写，如何修改，注意什么问题等

3. 使用tab的操作，实现缩进，默认整体向右移动，使用shift+tab，实现整体向左移动

4. 运算符和=两边习惯性加上一个空格

5. 源文件使用utf-8编码，在传统的dos窗口里，才设置为gbk编码

6. 行宽度不要超过80字符

7. 代码编写次行风格和行尾风格
